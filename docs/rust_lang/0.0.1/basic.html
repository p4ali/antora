Lang basics

refer to the main.rs and pm.rs

Module

Rest provide module system to hierachicallt split code into logical units (module), and manage the visibility (pub/private) between them.

A module is a collection of: functions, structs, traits, impl blocks and even other modules.

// A module named `my_mod`
mod my_mod {
    // Items in modules default to private visibility.
    fn private_function() {
        println!("called `my_mod::private_function()`");
    }

    // Use the `pub` modifier to override default visibility.
    pub fn function() {
        println!("called `my_mod::function()`");
    }

    // Items can access other items in the same module,
    // even when private.
    pub fn indirect_access() {
        print!("called `my_mod::indirect_access()`, that\n> ");
        private_function();
    }

    // Modules can also be nested
    pub mod nested {
        pub fn function() {
            println!("called `my_mod::nested::function()`");
        }
    }
}

fn main() {
    // Modules allow disambiguation between items that have the same name.
    function();
    my_mod::function();
}

Consume Crates

3rd party’s modules are normally published to Crates, and your own module can also colocate with your source code (locally in folders).

To consume 3rd party’s module, add it under the depdencies section of Cargo.toml, and use extern crate module_name in your source code:

Cargo.toml
...

[dependencies]
rand = "0.7.1"
my_mod = {path = "../mymod"}  # my module in local folder
main.rs
extern crate rand;
use rand::Rng;
extern crate my_mod;

fn main() {
    let mut rng = rand::thread_rng();
    let b:bool = rng.gen();
    println!("random boolean is {}", b);
    my_mod::function();
}

Run cargo publish to publish the mod to crates.io

Testing

Rust has build in support for unitest, to run the test: cargo test.

The test can be either embeded in the source code, or in a separate module.

To label a test,

lib.rs
pub mod greetings {
    pub mod english;
}

#[test]
#[should_panic]
#[ignore]
fn english_greeting_correct() {
    assert_eq!("hellos", greetings::english::hello());
}
