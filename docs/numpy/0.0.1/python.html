package, module distribution

  • Package - A folder that contains init.py file

  • Module - A python file with .py extension

  • Distribution - How one package relates to other packages and modules

To install a package named foo:
$ git clone https://github.com/user/foo
$ cd foo
$ python3.7 setup.py install
To use it without install foo:
# this will create symlinks to the source dir within site-packages instead of copying things
$ python3.7 setup.py develop
To find help:
$ python setup.py --help-commands

Python module

There are three different ways to define in Python:

  • A module can be written in Python itself

  • A module can be written in C and loaded dynamically at runtime, like the re(regular expression) module

  • A built-in module is intrinsically contained in the interpreter, like the itertools module

To write a module in Python is extremely straightforward - put it into a .py file

mymod.py
s="hello world"
a=[1, 2, 3]

def foo(arg):
  print(f'arg={arg}')

class Foo:
  pass

Several objects are defined in mymod.py: * s (a string) * a (a list) * foo (a function) * Foo (a class)

Assuming mymod.py is in a appropriate location (see next section), these objects can be accessed by importing the module as follows:

>>> import mymod
>>> print(mymod.s)
>>> mymod.a
>>> mymod.foo(['a', 'red', 'fox'])
>>> x=mymod.Foo()

The module search path

For statement:
import mymod

The interpreter will search: * The directory from which the input script was run or the current directory if the interpreter is beding run interactively * the list of directories containe in the PYTHONPATH environment variable, if it is set. (The format for PYTHONPATH is similar to PATH) * An installation-dependent list of directories configured at the time Python is installed

The sys.path variable contains above list, and can be accessed by:

>>> import sys
>>> sys.path
['', '/usr/local/Cellar/python/3.7.3/Frameworks/Python.framework/Versions/3.7/lib/python37.zip', '/usr/local/Cellar/python/3.7.3/Frameworks/Python.framework/Versions/3.7/lib/python3.7', '/usr/local/Cellar/python/3.7.3/Frameworks/Python.framework/Versions/3.7/lib/python3.7/lib-dynload', '/workspaces/github/python/venvs/numpy/lib/python3.7/site-packages', '/workspaces/github/python/venvs/numpy/src/requests']

>>> sys.path.append(r'/home/me/python/')

assuming /home/me/python/mymod.py is your module file.

Virtual environment and packages

To create virutal env
# this will create myenv folder if not exist, inside which containing a copy of Python interpreter, standard library, and various support files
$ python3 -m venv myenv

# to activate the myenv
$ source myenv/bin/activate
# and python will symlinked to real python interpreter, i.e., python3.7
$ python --version
Use pip to install, upgrade, and remove packages
$ pip search astronomy
$ pip install requests==2.6.0
$ pip install --upgrade request
$ pip show requests
$ pip remove astroobs
$ pip freeze > requirements.txt

# floating point format
pi = 3.141592653
print('%10.3f' % pi) #字段宽10，精度3, 3.142

# new line, by default, each print auto add new line, 0\n1\2...
for i in range(0,6):
  print(i)

# if you do not want new line: 0 1 2 3 4 5 6
for i in range(0,6):
  print(i, end=' ')
