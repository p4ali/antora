Uppercase means public

In go package, the uppercase var or funcs are public, and lower case is private which can not be accessed by other package.

Function

In go, function is first-class citizen, an be passed as value, no matter regular function func abc() or method func (x X) xxx(), or a closure func() {return func(){…​}}

Internal representation

// Func contains Type fields specific to func types.
type Func struct {
   Receiver *Type  // function receiver, can be nil or non-nil
   Results  *Type   // function results
   Params   *Type // function params
   Nname *Node   // function name
   // Argwid is the total width of the function receiver, params, and results.
   // It gets calculated via a temporary TFUNCARGS type.
   // Note that TFUNC's Width is Widthptr.
   Argwid int64
   Outnamed bool // can export or not

regular function

syntax:
func methodName([inputParamList]) [outputParamList] {
  body
}

Where the [] part is optional.

In case return multiple values, use tuple

func swap(x,y string) (string, string) {
  return y,x
}
a,b=swap("a","b")
Passing function as parameters
f := func() int {
  return 1
}

Method

Method is different from Function, but it also use func as keyword. The syntax

func (attachTo) methodName([inputParamList]) [outputParamList] {
  body
}

Where attachto can be either value TargetType or value *TargetType. The method will then be attached to the target type (struct). This can then turn the type to sth similar to Java class.

Closure

closure = func + context

Internally, golang will create a wrapper function, and golang will capture all free variables, and transform them into parameters(either value or ref) of the wrapper function.

Refer to golang closure internal for detail.

Pointer

syntax: var name *var-type, e.g.

var ip *int
var a int =1
ip = &a
fmt.Printf("pointer: %x\n", ip)
fmt.Printf("value: %d\n", *ip)

struct

To define a structure, use type and struct together, e.g.

type struct_name struct {
  member1 memberType
  member2 memberType

s := struct_name(value1, value2 ...)

  name string

var mInfo info

Anonymous struct

Package

The main entry is in package main. Normally the name of the package is same as the last segment of the path. e.g., import math.rand package, the package must start with package rand.

import package

// multiple import
import "fmt"
import "math"

// package then import
import (
  "fmt"
  "math"
)

Only the function/variables start with upper case can be accessed after importing.

main and package

golang is different from Java: * Any file in Java can have main method, and can be executed. * go can only have one main in each go file. The main can only be treated as execute entry iff under main package. * go build only build exec when it find the main method in the main package

package name vs folder name

best practice

  • package is folder

  • package name is folder name

  • package path is folder path

otherwise

  • same folder multiple package, compile error

  • folder name diff from package name, compile error, workaround is import alias path

go run, build, install

go run xxx.go // buld and run the binary, xxx.go must in main package and has main entry
go build xxx.go // build executable and save in current folder, xxx.go must in main package and has main entry
go install xxx.go // build executable and install to the predefined folder, normally $GOPATH/bin; and install dependecies into $GOPATH/pkg

create shared library

Similar to C, go allow you to build a package as shared library (.so for ddl, and .a for static). Before compile any shared library, the standard builtin packages should be installed as shared library. This will allow any other shared library to link with them. To do that

go install -buildmode=shared -linkshared std # will install builtin packages as shared library into $GOROOT/pkg/linux_amd64_dynlink/

Given go file:

package main

import "C"  // required for import C library
import "fmt"

//export Summ
func Summ(x, y int) int {
    return x + y
}

//export Hello
func Hello(str string) {
    fmt.Printf("Hello: %s\n", str)
}

func main() { // We need the main function to make possible, CGO compiler to compile the package as C shared library
}

Use following command to build library

# build exptest.go and output shared library libexptest.so
$ go build -buildmode=c-shared -o libexptest.so exptest.go
$ ls
  exptest.go  libexptest.h  libexptest.so

# build static library
$ go build -buildmode=c-archive -o libexptest.so exptest.go
jack@jack-VirtualBox:~/mygo/src/mylib$ ls
exptest.go  libexptest.h  libexptest.a

How to organize code

There doesn’t seem to be a standard way of organizing Go projects but How to write code specifies a best practice for most projects.

~/$GOPATH/
bin/
pkg/
src/
  github.com/
    username/
      mypack/
          foo.go
          bar.go
          mypack_test.go
      mylib/
          utillib.go
          utillib_test.go

By doing it this way, you can have a separate repository for mylib that can be used for other projects and can be retrieved by "go get". The mypack project can import the library using github.com/username/mylib.

In summary, all golang projects in the community can be categorized under * Libraries (no executable binaries) * Single Project (contains only 1 executable binary), such as etcd * Tooling Projects (contains multiple executable binaries), such as kubernetes, has multiple main function organized under a package cmd, the cmd/ package manages the number of binaries (tools) that we want to build.

~/$GOPATH/
bin/
pkg/
src/
  repository_remote/
    user_name/
      project_name/
        cmd/
          binary_one/
             main.go
          binary_two/
             main.go
          binary_three/
             main.go
        other_pkg/
