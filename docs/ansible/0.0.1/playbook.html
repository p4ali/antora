Playbook

  • Plays map hosts to tasks

  • A play can have multiple tasks

  • A playbook can have multiple plays

Playbook Breakdown

playbook.yml
---
- hosts: webservers       (1)
  remote_user: root
  sudo: yes
  gather_facts: no
  tasks:
  - name: Install Apache
    yum: name=httpd state=present
  - name: Start Apache
    service: name=httpd enabled=yes state=started

- hosts: dbservers        (2)
  remote_user: root
  vars:
    git_repo: https://gihub.com/repo.git
    db_name: wordpress
  tasks:
  - name: Install MySQL
    yum: name=mariadb-server state=present
  - name: Start MySQL
    service: name=mariadb enabled=yes state=started

where we have two plays,

1 The play of webservers
2 The play of dbservers

tasks

  • Tasks are exeuted in order - top down

  • Tasks use modules

Syntax of task
tasks:
- name: Name thsi task for readability
  <module>: <parmeters>=<go_here>

# example
- name: deploy Apache configuration file
  copy: src=/ansible/files/conf/httpd.conf dest=/etc/httpd/conf/

Excution of playbooks

ansible-playbook playbook.yml  —tags "web" —limit atlanta

It assume the inventory is defined inside ansible configuration file. Otherwise it need -i. If a host fails a task, that host is removed from the rest of the playbook execution.

Use tag can selectively run tasks or roles, limit can select hosts to run.

# site.yml

- include: webservers.yml tags=web
- include: dbservers.yml tags=db

Retrying failed host executions

ansible-playbook --limit@/home/me/ping.retry
