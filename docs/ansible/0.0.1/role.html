Roles and Galaxy

Roles provide a framework for fully independent, or interdependent collections of variables, tasks, files, templates, and modules.

Roles are the primary mechanism for breaking a playbook into multiple files, and compose them with include.

Possible roles example:

  • Wordpress

  • Mysql

  • Repository

  • Build

Each role follows a standard directory structure, see how to create a role for detail

To find role

Find role and download from Galaxy, follow the readme.

To install role

# download fubarhouse.golang to ~/.ansible/roles
ansible-galaxy install fubarhouse.golang

To use role

---
- hosts: localhost
  # sudo: yes
  gather_facts: yes
  vars:
    go_version: 1.12
    GOROOT: /home/vagrant/goroot
    GOPATH: /home/vagrant/go

   # ansible-galaxy install fubarhouse.golang
  roles:
  - fubarhouse.golang

  # TODO: factor this out to a file and include
  tasks:
  - name: Clone fortio
    git:
      repo: "https://github.com/fortio/fortio.git"
      dest: "{{GOPATH}}/src/fortio.org/fortio"
      clone: yes
      force: yes

  - name: Build fortio
    shell: |
      cd {{GOPATH}}/src/fortio.org/fortio && {{GOROOT}}/bin/go install
    environment:
      GOROOT: "{{GOROOT}}"
      GOPATH: "{{GOPATH}}"
